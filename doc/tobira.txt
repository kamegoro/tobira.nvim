1. INTRODUCTION                                             *tobira-introduction*

tobira.nvim watches your keystrokes and suggests the one command you are
missing, based on your actual usage patterns.

It runs silently in the background, tracks which commands you use, and when
it notices an inefficient pattern — like doing `dw` + `i` repeatedly when
`cw` exists — it waits until you are idle and quietly surfaces the better
path.

"Open the next door in your Vim journey."

2. SETUP                                                           *tobira-setup*

Install with lazy.nvim: >lua

  {
    "kamegoro/tobira.nvim",
    event = "VeryLazy",
    opts = {},
  }
<
Call `setup()` explicitly if not using lazy.nvim opts shorthand: >lua

  require("tobira").setup({
    lang = "en",
  })
<

3. COMMANDS                                                     *tobira-commands*

                                                                       *:Tobira*
:Tobira
  Show the next command suggestion immediately. Does not count toward the
  automatic suggestion cooldown.

                                                                 *:TobiraStats*
:TobiraStats
  Show a usage statistics panel. Leads with "Try these next" — efficiency
  gaps, commands you use heavily whose neighbors you have not yet
  discovered — since that is the one section that actually changes what
  you do. A mastery bar and your top commands follow. Total keystrokes and
  how many commands you have discovered sit in a quiet line at the bottom.

  Inside the stats panel:
    `g`   Close stats and open the guide panel
    `p`   Close stats and open the progress panel
    `q`   Close the panel

                                                                 *:TobiraGuide*
:TobiraGuide
  Toggle a dynamic cheatsheet panel. Content adapts to your current mastery
  level — commands you have already mastered are hidden, and pinned
  commands are always shown at the top. A command that was mastered but has
  gone quiet for the last two sessions reappears with a "forgotten" marker
  instead of staying hidden forever.

  Mastery markers in the guide panel:
    `●`          Pinned — always shown at the top
    `★`          Familiar, still shown because guide only lists commands
                 you have not fully mastered
    `☆`          Tried at least once
    (none, dim)  Never tried
    `⟳`          Forgotten — was practiced, has gone quiet

  Inside the guide panel:
    `p`   Toggle pin on the command under the cursor
    `x`   Toggle suppress on the command under the cursor
    `q`   Close the panel

                                                              *:TobiraProgress*
:TobiraProgress
  Open the skill tree panel. The header shows your current level and your
  overall `{n} / {total} mastered` ratio; each category heading shows its
  own `{done} / {total}`. Level-0 (never tried) commands render dimmed with
  no glyph. Move the cursor onto any command and a preview strip below the
  grid fills in with that command's usage sparkline, count, status, and how
  many uses remain to reach the next star.

  Mastery levels:
    `✗`     Suppressed (will not be suggested)
    (none)  Never used
    `☆`     Used at least once
    `★`     Used 100+ times
    `★★`    Used 1000+ times
    `★★★`   Used 5000+ times
    `⟳`     Forgotten — mastered, then quiet for the last 2 sessions
            (shown in the preview strip, not the grid glyph)
    `●`     Pinned — always shown, in both this panel and |:TobiraGuide|

  Inside the progress panel:
    `x`   Toggle suppress on the command under the cursor
    `p`   Toggle pin on the command under the cursor
    `g`   Close progress and open the guide panel
    `s`   Close progress and open the stats panel
    `q`   Close the panel

                                                                 *:TobiraReset*
:TobiraReset
  Reset all usage data. This clears the usage log stored at
  `~/.local/share/nvim/tobira/usage.json`.

4. CONFIGURATION                                                  *tobira-config*

Pass options to `setup()` or via the `opts` table in lazy.nvim.

Default configuration: >lua

  require("tobira").setup({
    lang               = "en",
    idle_delay         = 1500,
    idle_suggestions   = true,
    suggestion_cooldown = 300,
    max_shown          = 2,
  })
<

                                                              *tobira-config-lang*
`lang` (string, default: `"en"`)
  Display language. Supported values: `"en"`, `"ja"`.

                                                       *tobira-config-idle_delay*
`idle_delay` (number, default: `1500`)
  Milliseconds of keyboard inactivity before an automatic suggestion may
  appear. Increase this value if suggestions feel intrusive.

                                                  *tobira-config-idle_suggestions*
`idle_suggestions` (boolean, default: `true`)
  Whether to show suggestions automatically after idle. Set to `false` to
  disable automatic suggestions and use only |:Tobira| for manual triggering.

                                                *tobira-config-suggestion_cooldown*
`suggestion_cooldown` (number, default: `300`)
  Minimum seconds between automatic suggestions. Prevents repeated
  interruptions within a short period.

                                                       *tobira-config-max_shown*
`max_shown` (number, default: `2`)
  Maximum number of times the same command suggestion is shown per session.
  Resets to 0 on each Neovim startup.

5. USAGE                                                           *tobira-usage*

AUTOMATIC SUGGESTIONS ~

tobira monitors your keystrokes using |vim.on_key()|. It detects patterns
such as:

  - Repeating `f` without using `;` to repeat the motion
  - Using `dw` followed immediately by `i` instead of `cw`
  - Pressing `x` multiple times instead of `{n}x`
  - Using `0` then `w w w` instead of `^`

When a pattern is detected and you are idle for `idle_delay` milliseconds, a
suggestion appears in tobira's own floating window, near the top-right of the
screen. The window shows what you just did (when a specific pattern
triggered it), the command to try instead, and a short example. Its border
is colored by category (motion / edit / search / window / fold / mark /
macro) so a returning user can recognize the kind of suggestion at a glance.
The window auto-dismisses on its own after a few seconds — roughly
scaled to how much text it is showing — and never steals focus or blocks
input.

A muted hint at the bottom of the window always shows how to silence that
suggestion via |:TobiraProgress|, even when the window did not steal focus.

ADOPTION DETECTION ~

After a suggestion is shown, tobira watches whether you actually use the
suggested command. It does this passively — no button press required.
If you use the command, it is marked as adopted and will not be suggested
again once mastered (100+ cumulative uses).

The very first time you adopt a given suggestion, tobira shows a brief
separate confirmation ("Nice — you used ...") so the loop of noticing a
habit and improving it is not left completely silent. This only fires once
per command, ever.

SUPPRESSION ~

If a suggestion is not relevant to you, open |:TobiraProgress| and press
`x` on the command to suppress it permanently. Suppressed commands are
marked with `✗` and will never be suggested again unless you unsuppress them.

DATA STORAGE ~

Usage data is stored locally at: >

  ~/.local/share/nvim/tobira/usage.json
<
No data is sent externally. The file is updated on |VimLeave|.

6. DETECTED PATTERNS                                              *tobira-patterns*

Each pattern fires at most once per trigger sequence. After firing, the
suggestion is shown when you are next idle for `idle_delay` milliseconds.

MOTION ~

  `f{x}` → `f{x}` same line    Suggest `;`    — repeat the last f/t/F/T motion
  `l` × 5                       Suggest `w`    — jump forward by word
  `h` × 5                       Suggest `b`    — jump backward by word
  `w` × 5                       Suggest `W`    — jump forward by WORD
  `b` × 5                       Suggest `B`    — jump backward by WORD
  `j` × 5                       Suggest `{n}j` — use a count prefix instead
  `j` × 10                      Suggest `}`    — jump forward by paragraph
  `k` × 5                       Suggest `{n}k` — use a count prefix instead
  `k` × 10                      Suggest `{`    — jump backward by paragraph
  `k` (once) → `o`              Suggest `O`    — open new line above directly
  `0` → `w`                     Suggest `^`    — go to first non-blank
  `0` / `^` → `i`               Suggest `I`    — insert at first non-blank
  `$` → `a`                     Suggest `A`    — append at end of line
  `n` × 4                       Suggest `cgn`  — change next search match

EDIT ~

  `c$`                          Suggest `C`     — change to end of line
  `d$`                          Suggest `D`     — delete to end of line
  `dw` → `i`                    Suggest `cw`    — change word in one motion
  `yy` → `p`                    Suggest `yyp`   — duplicate the current line
  `dd` → `p`                    Suggest `ddp`   — swap current line with next
  `dd` → `i` / `a` / `o`       Suggest `cc`    — change whole line
  `dd` × 3                      Suggest `{n}dd` — delete N lines at once
  `D` → `i` / `a` / `o`        Suggest `C`     — change to end of line
  `x` (once) → `i` / `a` / `o` Suggest `s`     — substitute char (delete + insert)
  `x` × 3                       Suggest `{n}x`  — delete N chars at once
  `p` × 3                       Suggest `{n}p`  — paste N times
  `P` × 3                       Suggest `{n}P`  — paste above N times
  `~` × 3                       Suggest `{n}~`  — toggle case of N chars
  `.` × 3                       Suggest `{n}.`  — dot-repeat with a count
  `J` × 3                       Suggest `{n}J`  — join N lines at once
  `u` × 3                       Suggest `<C-r>` — redo instead of repeating undo
  `r{x}` × 3 (h/l between ok)  Suggest `R`     — enter Replace mode directly
  `>>` × 3                      Suggest `{n}>>` — indent N lines at once
  `<<` × 3                      Suggest `{n}<<` — dedent N lines at once
  `v` `i/a` `{obj}` `c/d/y`    Suggest direct operator (e.g. `ciw`, `di"`, `ya(`)
